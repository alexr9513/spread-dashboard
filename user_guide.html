Dashboard User Guide

Setup, navigation, and features

1 Setup

No installation required

The dashboard runs entirely in your browser. Simply open bond_spread_dashboard.html by double-clicking it — no Python, no server, no configuration files needed. Just load your data file when the overlay appears.

Supported file types

FormatExtensionNotes
Parquet.parquetRecommended for large datasets — fast and compact
Excel.xlsx / .xlsFirst sheet is used
CSV.csvComma-separated; headers required

Starting the dashboard

  • 1
    Double-click bond_spread_dashboard.html to open it in your browser. That's it — no server needed.
  • 2
    The Load Data File overlay opens automatically on first launch (or whenever no data has been loaded yet). Drop your file onto the drop zone or click BROWSE FILE.
  • 3
    After the file is read, a column mapping step appears. Match your file's columns to the four required fields:
    Required fieldWhat it expects
    Bond IDUnique bond identifier (ISIN, CUSIP, ticker, etc.)
    Spread (bps)Numeric spread in basis points (OAS, Z-spread, G-spread, etc.)
    Maturity DateBond maturity date (any standard date format)
    Snapshot DateDate of the data observation / pricing date
    Common column names are detected automatically — check the pre-selected values and adjust if needed.
  • 4
    Click LOAD DATA. The dashboard builds itself from your data.
Switching datasets
Click ⬆ LOAD FILE in the header at any time to load a different file. The overlay resets fully each time — column mappings are never carried over from a previous session.
Internet connection required
The dashboard loads Chart.js, SheetJS, and the Parquet reader from CDN on first open. Once cached by your browser, it works offline.

2 Settings Panel

Click ⚙ SETTINGS in the top-right corner to configure the dashboard. Changes are saved in your browser and persist across refreshes.

Sidebar filters

Filters are the groups on the left panel that let you drill down to specific bonds. You can add any column from your dataset as a filter.

StyleLookBest for
Search listSearch box + scrollable list with countsColumns with many values (issuers, tickers)
PillsClickable buttons in a rowColumns with few values (bond type, rating bucket)
Date listScrollable list + ADD TO CHART buttonDate columns — always put this last

Use the ▲ ▼ arrows to reorder. The All checkbox adds an "All" option that skips filtering on that step.

Filter order matters
Filters cascade: each step narrows the next. Put broad categories first (bond type, sector) and specific ones later (issuer, date). The last filter should always be a date list — it gets the ADD TO CHART button.

Chart axes

Check which numeric columns appear in the X / Y axis selectors on the chart. TTM (time to maturity) and the mapped spread column are always available.

Apply & Reset

APPLY & RELOAD saves your settings and rebuilds the dashboard. RESET DEFAULTS clears everything and auto-detects sensible defaults from your data.

3 Using the Dashboard

Selecting bonds

Work through the sidebar filters top to bottom. Each step narrows the next. As soon as all filters are set, a preview appears on the chart (white dots, dashed line).

Click + ADD TO CHART to confirm the selection. It gets its own colour and appears in the Active Series section at the top of the sidebar. You can add multiple series (e.g. same issuer at different dates, or two issuers on the same date).

Chart modes

ButtonWhat it does
SCATTERPlain dots — no curve fitting
FIT CURVEFits a curve through the bonds. Method chosen via ⚙ panel
⚙Opens the fitting method panel (Auto, Linear, Polynomial, Nelson-Siegel, Spline)

The X and Y axis selectors let you switch between TTM, duration, yield, or any numeric column you enabled in settings.

Zooming and panning

Scroll wheel zooms in and out on both axes simultaneously. Click and drag pans the chart in any direction.

A ↺ ZOOM button appears in the chart controls as soon as you zoom or pan. Click it to snap back to the original full view. The zoom resets automatically whenever the chart is re-rendered (e.g. when adding or removing a series).

Interacting with the chart

Hover over a dot to see its identifier, X value, and Y value in a tooltip.

Click a row in the bottom table to highlight that bond on the chart (larger dot, white fill). Click again to un-highlight.

Bottom table

The table shows all bonds for the current selection(s), sorted by TTM. Click ⊞ COLUMNS to show or hide individual columns. All columns from your dataset are available.

Stat bar

The bar at the top of the main panel shows summary statistics for the currently visible bonds: issuer name (or count if multiple), unique bond count, average / min / max spread, and average maturity.

Active series

Each added series appears as a coloured tag at the top of the sidebar. Click × to remove one, or clear all to start fresh.

4 Saved Views

A saved view captures everything about your current chart state: the active series, chart type, axis selection, curve fitting settings, and the current sidebar filter selections.

Saving a view

Click 💾 SAVE in the sidebar and give the view a name. The view is stored in your browser's localStorage.

Loading a view

Click 📂 VIEWS to open the saved views panel, then click a view name. The series and chart settings are always restored. The sidebar filters (which issuer, type, date were selected) are also restored automatically — but only if the currently loaded dataset is the same one the view was saved with. If you load a different file, only the series are restored and the filters reset to defaults.

Exporting and importing views

Use ↓ to export all saved views to a .json file and ↑ to import them back. This lets you share views between machines or back them up.

Views are browser-local
Views are stored in localStorage and are not shared between browsers or devices unless you export and import the JSON file.

5 Curve Fitting

Click FIT CURVE to fit a spread curve, then click the ⚙ gear icon to choose the fitting method and tweak parameters.

MethodWhen to use
AutoPicks the best method based on bond count (recommended)
LinearQuick baseline, very few bonds
Polynomial3–5 bonds, mild curvature. Tuneable degree (2–4)
Nelson-Siegel5+ bonds, economic interpretation (β₀, β₁, β₂, τ). Can fix τ
Spline6+ bonds, flexible shape, handles outliers. Tuneable smoothing (λ) and knots

The legend on the chart shows the method used, key parameters, and RMSE (fit quality in bps). The shaded band is ±1 RMSE — bonds outside this band are strong cheap/rich candidates.

Detailed reference
Click ? FITTING in the header for the full mathematical reference with formulas, parameter tables, and RMSE benchmarks.

7 Tips

Compare issuers: Add issuer A on date X, then add issuer B on the same date. Both curves appear on the chart with different colours.

Track an issuer over time: Add the same issuer on multiple dates. Each date gets its own colour — you can see how the curve shifted.

Spot cheap/rich bonds: Switch to FIT CURVE mode. Bonds above the curve (positive residual) trade wider than peers → potentially cheap. Bonds below → potentially rich. Use scroll wheel to zoom into the area of interest.

Senior vs Sub: If your data has a type column, add it as a pills filter in settings. Then compare Senior and Subordinated curves separately — mixing them distorts the fit.

Export the chart: Right-click the chart canvas → "Save image as…" to get a PNG.

Too many columns in settings? All axes are unchecked by default. Only check the ones you actually want as X/Y options.

Browser data: Your filter settings, axis configuration, and saved views are stored in your browser's localStorage. Clearing browser data resets everything to defaults.

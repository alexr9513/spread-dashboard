Dashboard User Guide

Setup, navigation, and features

First-time setup

1. Install Python dependencies:

$ pip install pandas pyarrow

2. Open config.json and set 2 things:

input_filePath to your data file (.parquet, .csv, or .xlsx)
columns_essentialMap your column names to the 4 required fields: bond ID, spread, maturity date, snapshot date

3. Run the export script:

$ python dashboard.py

4. Start a local web server:

$ python -m http.server 8000

5. Open in your browser:

http://localhost:8000/bond_spread_dashboard.html
Updating data
When your data changes, just re-run python dashboard.py and hard-refresh the browser (Ctrl + Shift + R).

2 Settings Panel

Click ⚙ SETTINGS in the top-right corner to configure the dashboard. Changes are saved in your browser and persist across refreshes. No need to edit config files or re-run Python.

Sidebar filters

Filters are the groups on the left panel that let you drill down to specific bonds. You can add any column from your dataset as a filter.

StyleLookBest for
Search listSearch box + scrollable list with countsColumns with many values (issuers, tickers)
PillsClickable buttons in a rowColumns with few values (bond type, rating bucket)
Date listScrollable list + ADD TO CHART buttonDate columns (always put this last)

Use the ▲ ▼ arrows to reorder. The All checkbox adds an "All" option that skips filtering on that step.

Important
Filters cascade: each step narrows the next. Put broad categories first (bond type, sector) and specific ones later (issuer, date). The last filter should always be a date list — it gets the ADD TO CHART button.

Chart axes

Check which numeric columns appear in the X / Y axis selectors on the chart. TTM (time to maturity) and OAS (spread) are always available.

Apply & Reset

APPLY & RELOAD saves your settings and rebuilds the dashboard. RESET DEFAULTS clears everything and auto-detects sensible defaults from your data.

3 Using the Dashboard

Selecting bonds

Work through the sidebar filters top to bottom. Each step narrows the next. As soon as all filters are set, a preview appears on the chart (white dots, dashed line).

Click + ADD TO CHART to confirm the selection. It gets its own colour and appears in the Active Series section at the top of the sidebar. You can add multiple series (e.g. same issuer at different dates, or two issuers on the same date).

Chart modes

ButtonWhat it does
SCATTERPlain dots — no curve fitting
FIT CURVEFits a curve through the bonds. Method chosen via ⚙ panel
⚙Opens the fitting method panel (Auto, Linear, Polynomial, Nelson-Siegel, Spline)

The X and Y axis selectors let you switch between TTM, Duration, Yield, or any numeric column you enabled in settings.

Interacting with the chart

Hover over a dot to see its ISIN, X value, and Y value in a tooltip.

Click a row in the bottom table to highlight that bond on the chart (bigger dot, white fill). Click again to un-highlight.

Bottom table

The table shows all bonds for the current selection(s). Columns are sorted by TTM.

Click ⊞ COLUMNS to show/hide columns. All columns from your dataset are available — check the ones you want to see.

Stat bar

The bar at the top of the main panel shows summary statistics for the currently visible bonds: issuer name (or count if multiple), unique bond count, average / min / max OAS, and average maturity.

Active series

Each added series appears as a coloured tag at the top of the sidebar. Click × to remove one, or clear all to start fresh.

4 Curve Fitting

Click FIT CURVE to fit a spread curve, then click the ⚙ gear icon to choose the fitting method and tweak parameters.

MethodWhen to use
AutoPicks the best method based on bond count (recommended)
LinearQuick baseline, very few bonds
Polynomial3–5 bonds, mild curvature. Tuneable degree (2–4)
Nelson-Siegel5+ bonds, economic interpretation (β₀, β₁, β₂, τ). Can fix τ
Spline6+ bonds, flexible shape, handles outliers. Tuneable smoothing (λ) and knots

The legend on the chart shows the method used, key parameters, and RMSE (fit quality in bps). The shaded band is ±1 RMSE — bonds outside this band are strong cheap/rich candidates.

Detailed reference
Click ? GUIDE in the header for the full mathematical reference with formulas, parameter tables, and RMSE benchmarks.

5 Data Preprocessing

If you need to clean or transform your data before it reaches the dashboard, open dashboard.py and edit the preprocess() function. It receives the raw DataFrame with original column names and returns the cleaned version.

Common examples:

# Filter to EUR only
df = df[df["Currency"] == "EUR"]

# Drop columns you don't need
df = df.drop(columns=["col_a", "col_b"])

# Clean a text column
df["Rating"] = df["Rating"].str.strip().str.upper()

# Clip negative spreads
df["OAS vs Govt"] = df["OAS vs Govt"].clip(lower=0)

# Create a derived column
df["Bucket"] = pd.cut(df["OAS vs Govt"], bins=[0,200,400,800,9999],
    labels=["Tight","Mid","Wide","Distressed"])

After editing, re-run python dashboard.py and refresh the browser.

6 Tips

Compare issuers: Add issuer A on date X, then add issuer B on the same date. Both curves appear on the chart with different colours.

Track an issuer over time: Add the same issuer on multiple dates. Each date gets its own colour — you can see how the curve shifted.

Spot cheap/rich bonds: Switch to FIT CURVE mode. Bonds above the curve (positive residual) trade wider than peers → potentially cheap. Bonds below → potentially rich.

Senior vs Sub: If your data has a type column, add it as a pills filter in settings. Then compare Senior and Subordinated curves separately — mixing them distorts the fit.

Export the chart: Right-click the chart canvas → "Save image as…" to get a PNG.

Too many columns in settings? All axes are unchecked by default. Only check the ones you actually want as X/Y options.

Browser data: Your filter and axis settings are stored in your browser's localStorage. Clearing browser data or using a different browser resets to defaults. Your bond data in data.json is never modified.
